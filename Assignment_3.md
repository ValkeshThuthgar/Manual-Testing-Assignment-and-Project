### Task 2: SDLC Phase Mapping for Zomato
1. Requirement Gathering: Team plans Zomato Gold Eco-friendly Delivery feature.
2. System Design: Architects create database and API flow for subscription screens.
3. Coding: Developers write frontend and backend code for the checkout button.
4. Testing: QA executes functional and UI testing on mobile devices to check bugs.
5. Deployment: The final build is released on Play Store and App Store.
6. Maintenance: Monitoring server performance and pushing quick hotfixes.

### Task 3: V-Model Testing Activities for Swiggy/Zomato

2. System Design -> System Test Design: Preparing end-to-end user checkout scenarios.
3. Architecture Design -> Integration Test Design: Verifying Cart Module and Payment API.
4. Coding -> Unit Testing: Checking code functions like calculateGST output.

### Task 4: Methodology Comparison
* V-Model Strengths: Early testing, high discipline, easy tracking.
* V-Model Weaknesses: Highly rigid to changes, late software build/demo.
* RAD Model Strengths: Rapid feedback, high flexibility, reduced cycle time.
* RAD Model Weaknesses: Heavy testing volatility, high dependency on skills.
