1,"What are the main pillars of OOPs, and how do they apply to Selenium?","The four main pillars are Inheritance, Polymorphism, Encapsulation, and Abstraction.","In automation, Inheritance is used for Base Classes (reusing WebDriver instances). Polymorphism allows handling different browsers. Encapsulation is used in Page Object Model (POM) to hide web elements. Abstraction hides the internal implementation of framework utilities."
2,What is Class and Object in Java? How do you use them in Selenium?,"A Class is a blueprint/template, and an Object is an instance of that class.","A Page Class (e.g., LoginPage.java) acts as a blueprint defining elements and actions. When you write LoginPage login = new LoginPage(driver);, you create an object to actually interact with that webpage."
3,Explain Inheritance with a real-time Selenium framework example.,Inheritance is a mechanism where one class acquires the properties and behaviors of a parent class using the extends keyword.,We create a BaseTest class that contains setup (driver.init()) and teardown (driver.quit()) methods. All test classes then extends BaseTest to automatically inherit and use the WebDriver instance without rewriting setup code.
4,What is Polymorphism and what are its types?,"Polymorphism means ""many forms"". It allows an object to behave differently in different situations. It has two types: Compile-time (Overloading) and Runtime (Overriding).",Compile-time: Using the same method name with different parameters.Runtime: Overriding a parent class method in a child class to change its default behavior.
5,How do you use Method Overloading in Selenium automation?,Method Overloading (Compile-time Polymorphism) occurs when multiple methods in the same class have the same name but different parameters.,"We use overloading for wrapper functions like click(). For example, click(WebElement element) to click directly, or click(String xpath) to locate and click, or type(WebElement element, String text) vs type(WebElement element, int numbers)."
6,Give a practical example of Method Overriding in Selenium.,Method Overriding (Runtime Polymorphism) happens when a child class provides a specific implementation for a method already defined in its parent class.,"If your BasePage has a generic waitForElement() method, but a specific DashboardPage requires a completely different wait strategy or a higher timeout for a heavy chart element, you override waitForElement() in the DashboardPage class."
7,What is Encapsulation and how is it implemented in Page Object Model (POM)?,"Encapsulation is data hiding. It binds variables (data) and methods (code) together into a single unit, keeping fields private and exposing them via public methods.","In POM, we declare WebElements as private WebElement username; at the top of the class. We do not expose these elements directly to test scripts. Instead, we create a public void login(String user) method to interact with them, preventing accidental test script modifications."
8,What is Abstraction and how does WebDriver itself exhibit it?,Abstraction is the process of hiding internal implementation details and showing only essential functionality to the user.,"WebDriver is an interface. When you write WebDriver driver = new ChromeDriver();, you only call abstract methods like get() or findElement(). You don't know how Google or Mozilla wrote the background code to physically open the browser; you just use the exposed actions."
9,What is the difference between an Interface and an Abstract Class?,An Interface can only have abstract methods (until Java 8) and public final variables. An Abstract Class can have both abstract (without body) and concrete (with body) methods.,"Use an Interface (like WebDriver) when you want to define a strict contract for all browsers to follow. Use an Abstract Class (like a BasePage) when you want to provide shared, fully coded helper methods along with a few forcing abstract rules."
10,Can we create an object of an Interface? Why do we write WebDriver driver = new ChromeDriver();?,"No, you cannot create an object of an interface directly (e.g., new WebDriver(); is invalid).","WebDriver is an interface. ChromeDriver is a concrete class implementing that interface. Writing it this way allows you to achieve Runtime Polymorphism, making it incredibly easy to switch your execution to new FirefoxDriver() or new EdgeDriver() down the road."
11,"What is a Constructor, and why do we use it in Page Objects?",A Constructor is a special block of code automatically invoked when an object is created. It has the same name as the class and no return type.,"In POM, we use constructors to pass and initialize the WebDriver instance from the running test case to the page class using the this keyword:public LoginPage(WebDriver driver) { this.driver = driver; }"
12,What is the purpose of the this keyword in Java?,this is a reference variable that refers to the current class instance/object.,"It resolves ambiguity when instance variables and constructor parameters share identical names. For example, assigning the driver instance passed from the test framework: this.driver = driver;."
13,What is the purpose of the super keyword in Java?,"super is a reference variable used to refer to immediate parent class objects, methods, or constructors.","If your parent page constructor requires initialization parameters (like a driver configuration), you use super(driver); as the very first line in your child page class constructor to run the parent's logic first."
14,What is the difference between this() and super()?,"this() invokes a constructor from the same class (constructor chaining), while super() invokes the constructor of the immediate parent class.","Both must be the absolute first statement inside a constructor execution block, meaning you cannot use both this() and super() inside the exact same constructor."
15,Can a constructor be private? How is it useful in automation frameworks?,"Yes, a constructor can be marked private. This prevents the class from being instantiated from anywhere outside of itself.",This is the foundation of the Singleton Design Pattern. We use it for configuration managers or database readers where we only want one single instance of a reader globally across the entire test suite run.
16,What is the static keyword in Java? Can we make a WebDriver instance static?,"static means the variable or method belongs directly to the class itself, rather than to instances (objects) of the class.","Yes, you can make WebDriver static, which makes it easy to access across pages without passing it via constructors. However, avoid this if you plan to run tests in parallel, as threads will overwrite each other's static driver instance."
17,What are the limitations of using static drivers in parallel testing?,Static fields are shared across all objects and threads executing within the same Java Virtual Machine (JVM) memory space.,"If Thread A opens Chrome and Thread B opens Firefox using a shared static driver variable, Thread B will hijack or corrupt Thread A's execution window, causing tests to crash randomly."
18,What is ThreadLocal and how does it fix the static driver issue?,"ThreadLocal is a Java class that provides thread-local variables, giving each individual thread its own isolated copy of a variable.","Instead of public static WebDriver driver;, we write public static ThreadLocal<WebDriver> driver = new ThreadLocal<>();. This keeps your driver globally accessible across your framework while remaining perfectly safe for parallel test execution."
19,What is the final keyword? Can we override a final method?,"final is a restriction keyword. A final variable cannot be reassigned, a final method cannot be overridden, and a final class cannot be inherited.","No, you cannot override it. If you build a framework utility method like readExcelConfig() and want to ensure nobody down the line accidentally overrides or alters its core behavior, you mark it final."
20,"What is the difference between final, finally, and finalize?",final is an access modifier for restriction. finally is a block used with try-catch for cleanup. finalize is a legacy method used by the Garbage Collector.,"final: Fixes variable values or prevents overriding.finally: Always executes after try-catch, used for driver.quit() or closing DB connections to prevent memory leaks regardless of test success/failure."
